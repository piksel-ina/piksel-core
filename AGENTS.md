# AGENTS.md

## What this repo is

A Docker-based satellite data management platform built on Open Data Cube (ODC) + PostGIS for the Indonesia region. There is no root-level Python package — the repo is infrastructure, config, product definitions, and Makefile automation. All services run in Docker.

## Setup (required before anything works)

```bash
make setup    # creates .env from .env.example, generates datacube.conf
make build    # builds ODC Docker image
make up       # starts PostgreSQL + ODC containers
make init-db  # initializes ODC database schema (run once after first `up`)
```

Services are in `docker/docker-compose.yml`. Optional services (jupyter, explorer) use Docker Compose profiles and are excluded from `make up` by default.

## Key paths

- `docker/docker-compose.yml` — main compose file (not at repo root)
- `docker/docker-compose-test.yml` — isolated test compose (project name `piksel-test`)
- `docker/odc/Dockerfile` — ODC image; uses `uv` with `pyproject.toml` (pinned `datacube==1.9.10`)
- `docker/odc/pyproject.toml` — ODC Python dependencies
- `products/*.yaml` — ODC product definitions, mounted into containers at `/home/venv/products/`
- `metadata/` — custom ODC metadata types
- `scripts/generate_config.sh` — runs `envsubst` on `datacube.conf.template`
- `datacube.conf` — generated file, gitignored; template is `datacube.conf.template`

## Testing

Tests require Docker. A `.venv` on the host is used to run pytest against containers.

```bash
make test-deps          # creates .venv, installs pytest + deps (run first)
make test-unit          # unit tests only (no Docker needed)
make test               # starts test Docker stack, runs all tests, tears down
make test-integration   # integration tests only (starts test stack)
make test-verbose       # all tests with -xvs
make test-clean         # removes .venv, caches, test containers/volumes
```

Integration tests wait for the `piksel-test-odc-1` container to report healthy before running (see `tests/conftest.py`).

Test structure: `tests/unit/` and `tests/integration/`. Test fixtures use `@pytest.mark.dependency()` for ordered execution.

## Dependency management

Uses `uv` (astral) for compiling and syncing Python dependencies inside Docker images:

```bash
make compile-deps       # compiles both ODC + Jupyter lockfiles
make update-deps        # recompiles with --upgrade
```

Source files: `docker/odc/pyproject.toml`, `docker/jupyter/requirements.in`.

## Common pitfalls

- **`.env` must exist** before `make up`, `make test`, or any compose command. Created by `make setup` from `.env.example`.
- **`datacube.conf` is gitignored** — generated by `make setup` via `envsubst`. Don't edit it directly; edit `datacube.conf.template`.
- **Products must be added to ODC after init**: `make all-products` or `make add-product F=<file.yaml>`.
- **Indexing commands** run `stac-to-dc` or `s3-to-dc` inside the ODC container — the stack must be running.
- All Docker Compose commands are wrapped in the Makefile. The compose file path is `docker/docker-compose.yml` (not the default `docker-compose.yml` at root).
- Docker platform is pinned to `linux/x86_64`.
- Python version: 3.12.

## CI (GitHub Actions)

- **Test workflow** (`.github/workflows/test-odc.yml`): runs on push to `main`/`develop`/`feature/*` and PRs to `main`/`develop`. Runs `make test-deps` then `make test`.
- **Tag-based builds** (`.github/workflows/tag-build.yml`): pushes tags matching `odc-*`, `jupyter-*`, `dev-jupyter-*` trigger Docker builds pushed to AWS ECR.
- **Manual build** (`.github/workflows/manual-build.yml`): workflow dispatch to build any component.
